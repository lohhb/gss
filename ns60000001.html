Technical Support Notes

Subject: Error F5041 Torque error in Low Power state

Handler Model: NS-6040 (UTAC, NS60-15T)

Controller: SRC520CE

Date: 14 Nov 2007


 

Symptom

NS-6040 Output Arm X Axis experience Error 5041 Torque error in Low Power state after motor replacement.

Torque error could be due to the wrong setting of distance between pulley to motor shaft depth.


 

Action

During MCORG 1, X-Axis move towards X -Axis Home sensor position and hit hard stopper.

Use metal plate to block sensor, to check whether sensor is working or not, found out that X-Axis home sensor one wire broken, after solder back the broken wire able to execute MCORG 1 and MCAL.


 

Did point teaching for Auto1, Auto2 but Auto3 and Fix3 unable to do teaching because limit error.

Move X-Axis arm to X-Axis Home sensor such that sensor red LED just turn on, then perform the following:

>selrb 3

>motor on

>calpls 0,0,0,0

>calib 1

>mcal


 

Peform teaching on Auto1 and Auto3 first to ensure no limit error.


 

Remarks:

Check for the following if there is Torque error in low Power state:

- loose connection of signal connector behind the Drive Unit

- MCD file of robot eg. MCSys03.MCD

- Belt tension

- any dummy inertial weight missing from pulley

- faulty motor

- faulty DMB

- Motor Power Unit ( AC/DC Board) in Drive Unit

- Solid State Relay (SSR), confirm whether able to move robots that are link to that particular DU

- wrong power setting, CT Gain, CT Offset in SPEL

- wrong power rating of motor power module


 

Cause

Output Arm's X- Axis Home sensor's wire broken.
